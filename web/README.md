# SydneyTrips web (Next.js)

The SydneyTrips frontend: trips CRUD, the map-based planning canvas with the
Pareto carousel, the live driver and passenger tracking views, and the
cost-split breakdown. No login — the API stamps an anonymous `trips_session`
cookie that owns the browser's trips.

## Stack

- Next.js 16 (App Router, Turbopack, React 19)
- TypeScript strict mode
- Tailwind v4 + shadcn/ui (neutral base, CSS variables)
- TanStack Query for server cache, Zustand for UI state
- Google Maps via `@vis.gl/react-google-maps`
- Zod + react-hook-form for form validation
- Vitest + Testing Library + Playwright

## Getting started

```bash
# create web/.env.local with at least:
#   API_BASE_URL=http://localhost:5000
#   NEXT_PUBLIC_API_BASE_URL=http://localhost:5000
# and optionally NEXT_PUBLIC_GOOGLE_MAPS_KEY=<browser key> for a real basemap
npm install
npm run dev
```

The dev server listens on `http://localhost:3000`.

## Environment

| Var                       | Required | Notes                                                                 |
| ------------------------- | -------- | --------------------------------------------------------------------- |
| `NEXT_PUBLIC_API_BASE_URL`| yes      | Public URL of the Trips API (used by client-side preview helpers).    |
| `API_BASE_URL`            | no       | Server-side override for the proxy. Defaults to `NEXT_PUBLIC_API_BASE_URL`. |
| `NEXT_PUBLIC_GOOGLE_MAPS_KEY`| no    | Browser-side Google Maps key. The map degrades to the SVG fallback canvas when missing. |

There's no auth config: the `trips_session` cookie is minted and owned by the API, so the web app holds no secret and signs nothing.

`gen:api` is a graceful no-op when the OpenAPI spec hasn't shipped yet.

## API integration

There's no token to manage. Requests flow through the Next.js route-handler
proxy at `/api/proxy/[...path]/route.ts`, which forwards the browser's cookies
(including the httpOnly `trips_session` cookie the API mints) straight to the
upstream Trips API and mirrors any `Set-Cookie` back. No JWT, no auth gate — the
API decides what the cookie grants.

- `src/lib/api/schema.ts` — hand-written request/response types; the app reads from these.
- `src/lib/api/types.ts` — full OpenAPI types generated by `npm run gen:api` from `src/Trips.Api/openapi.json` (used where the exhaustive shape is needed).
- `src/lib/api/client.ts` — fetch wrapper with `ApiError` + JSON typing.
- `src/lib/api/hooks.ts` — TanStack Query hooks.
- `src/lib/api/adapters.ts` — translates API DTOs into the UI's nested-LatLng types.

Regenerate `types.ts` with `npm run gen:api` whenever the API contract changes (see the root `CLAUDE.md`).

## Map layers

`src/components/plan/PlanMap.tsx` renders these overlay groups via `@vis.gl/react-google-maps`:

| Overlay           | Type     | Style                                                                          |
| ----------------- | -------- | ------------------------------------------------------------------------------ |
| candidate nodes   | marker   | small grey dot — every candidate PT pickup node.                               |
| driver routes     | polyline | road-snapped per-driver line (Okabe-Ito palette), white casing, fanned apart where routes share roads. |
| passenger legs    | polyline | dashed home→pickup legs coloured by TfNSW mode (walk = slate), with a minute/mode chip at the midpoint. |
| pickup stops      | marker   | green pin (`#34A853`), plus a TfNSW mode chip for transit hubs.                |
| origins           | marker   | name-labelled dot coloured to match the driver they ride with; drivers larger.|
| destination       | pin      | red Google pin anchored at the tip.                                            |

The map degrades to the SVG `MapFallback` canvas — drawn from the same real coordinates — when `NEXT_PUBLIC_GOOGLE_MAPS_KEY` is unset.

- `/` — redirects straight to `/trips` (no login step).
- `/trips` — list this browser's trips.
- `/trips/new` — create-trip form with geocode preview.
- `/trips/[id]` — overview, participant CRUD.
- `/trips/[id]/plan` — full-screen map + right-hand panel: weight sliders,
  optimise button, Pareto carousel.
- `/trips/[id]/driver` — live driver manifest + route polyline (SignalR).
- `/trips/[id]/passenger` — live passenger ETA view (SignalR).
- `/trips/[id]/cost` — cost-split breakdown.

`proxy.ts` (Next 16's renamed `middleware.ts`) is a no-op: with auth gone there's
nothing to gate — every browser carries an anonymous `trips_session` cookie minted
by the API on its first call. The file is kept so the matcher stays explicit.
